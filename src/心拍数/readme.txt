取得した心拍数データの内容を解説します。このデータはGarmin Connect APIを通じて取得されたもので、以下のような情報を含んでいます：

userProfilePK: ユーザープロフィールの一意な識別子です。
calendarDate: データが取得されたカレンダー上の日付です。
startTimestampGMT と endTimestampGMT: データが記録された開始と終了の時刻（グリニッジ標準時）です。
startTimestampLocal と endTimestampLocal: データが記録された開始と終了の時刻（ローカルタイムゾーン）です。
maxHeartRate: 記録された期間中の最大心拍数です。
minHeartRate: 記録された期間中の最小心拍数です。
restingHeartRate: 休息時の心拍数です。
lastSevenDaysAvgRestingHeartRate: 過去7日間の平均休息時心拍数です。
heartRateValues: 心拍数の値のリストです。各エントリーはタイムスタンプ（ミリ秒単位のUNIX時間）と心拍数のペアで構成されています。
heartRateValueDescriptors: 心拍数値リストの各項目が何を意味するかを説明するキーのリストです。
heartRateValuesの各エントリーは、特定の時刻における心拍数を示しています。例えば、[1701356400000, 52]というエントリーは、UNIX時間の1701356400000（ミリ秒単位）における心拍数が52であることを意味しています。UNIX時間はエポック（1970年1月1日0時0分0秒UTC）からの経過ミリ秒数であり、これを日時に変換することで、具体的な時刻を知ることができます。

このデータは、ユーザーの心拍数を時間とともに追跡するのに役立ち、フィットネスの監視、健康管理、トレーニングの効果分析などに使用できます。また、最大心拍数、最小心拍数、平均休息時心拍数などの情報は、ユーザーの心臓の健康状態を評価するための重要な指標となります。ｓ
